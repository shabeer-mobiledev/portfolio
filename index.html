Recipe App

A mobile app that allows users to search and view recipes. It leverages Retrofit to fetch data from external APIs, Firebase for storing user data, Jetpack Compose for building a modern and responsive UI, and Dagger Hilt for Dependency Injection to manage app components efficiently.

  • Retrofit: Used for making network requests to external recipe APIs. Retrofit simplifies the process of converting JSON data into Kotlin objects, making API communication seamless and efficient.
  • Firebase: Firebase is used for user authentication, data storage, and syncing saved recipes in real-time, allowing users to access their saved recipes from any device.
  • Jetpack Compose: The UI is built using Jetpack Compose, allowing for a declarative, modern, and efficient UI design with minimal boilerplate code.
  • MVVM Architecture: The app follows the MVVM (Model-View-ViewModel) pattern, ensuring a clean separation of concerns and making the app more maintainable and testable.
  • LiveData & ViewModel: LiveData is used to observe changes in the data, and the ViewModel manages UI-related data in a lifecycle-conscious manner, ensuring consistency across configuration changes.
  • Dependency Injection (Dagger Hilt): Dagger Hilt is used for Dependency Injection, making it easier to manage app components, promoting loose coupling, and improving code modularity. It simplifies the injection of dependencies into various parts of the app, such as Retrofit and Firebase services.
  • Material Design: Implements Material Design components for a user-friendly and visually appealing interface that provides a consistent user experience across Android devices.
